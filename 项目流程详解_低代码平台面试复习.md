# Vue3 低代码平台：项目流程详解

> 项目目录：`D:\Vue3_低代码平台`
> 目标：让没学过低代码编辑器的人也能看懂完整流程，并可用于前端面试复习。

---

## 1. 一句话理解项目

这是一个"可视化网页编辑器"，类似简化版的 Wix 或 Webflow，用户通过拖拽组件就能生成营销活动页面。

### 1.1 什么是营销页面？

营销页面是指用于商业推广和用户转化的专题页面：

| 类型 | 示例场景 |
| --- | --- |
| 活动页 | 双11大促、618购物节、周年庆典 |
| 落地页 | 新用户拉新、产品发布、限时优惠 |
| 专题页 | 品牌宣传、新品首发、节日营销 |

**营销页面特点**：
- 上线频繁
- 样式要求灵活
- 页面变化快
- 技术不希望每次都手写代码

**传统做法**：前端每次写一个新页面
**这个平台的做法**：运营拖拽组件 → 自动生成页面

### 1.2 用户在页面里做这些事

1. 从左侧拖一个组件（文本、按钮、图表等）到画布
2. 在画布中移动、缩放、旋转组件
3. 在右侧修改样式、配置动画、绑定事件
4. 用工具栏保存、预览、导入导出 JSON、撤销重做、截图

**本质**：它把"写代码搭页面"变成"拖拽搭页面"。

## 3. 核心设计思想

### 3.1 页面数据结构设计（最关键）

**核心思想**：页面不是 DOM，而是一棵 JSON 树。

采用 JSON Schema 描述页面结构与组件树，通过 TypeScript 接口与枚举约束组件属性与交互行为，保障系统的类型安全与可扩展性。

```typescript
// 页面数据结构示例
{
  "id": "page_001",
  "title": "双11活动页",
  "components": [
    {
      "id": "text_001",
      "component": "VText",
      "label": "标题文本",
      "style": {
        "position": "absolute",
        "top": 100,
        "left": 200,
        "width": 300,
        "height": 50,
        "fontSize": 24,
        "color": "#333333"
      },
      "propValue": "限时优惠",
      "animations": [],
      "events": {},
      "linkage": { duration: 0, data: [] }
    },
    {
      "id": "image_001",
      "component": "Picture",
      "style": { "top": 200, "left": 100, "width": 400, "height": 300 },
      "propValue": { "url": "https://example.com/banner.jpg", "flip": { "horizontal": false, "vertical": false } }
    }
  ]
}
```

**渲染原理**：Vue 通过 `<component :is="componentMap[item.component]" v-bind="item" />` 动态渲染组件，实现数据驱动视图。

**这本质是**：编辑态是 JSON，运行态也是 JSON 驱动渲染。

### 3.2 命令模式与撤销重做

将用户操作抽象为命令对象并维护历史栈，完整支持组件新增、删除及属性变更等高频编辑场景。

```typescript
// 快照管理核心逻辑（实际实现）
// 最大快照数量限制，防止内存溢出
const MAX_SNAPSHOTS = 50

// 记录快照
recordSnapshot(): void {
  // 限制快照数量
  if (this.snapshotData.length >= MAX_SNAPSHOTS) {
    this.snapshotData.shift()
    this.snapshotIndex--
  }
  // 添加新的快照
  this.snapshotData[++this.snapshotIndex] = deepCopy(this.componentData)
  // 在 undo 过程中，添加新快照时清理后面的快照
  if (this.snapshotIndex < this.snapshotData.length - 1) {
    this.snapshotData = this.snapshotData.slice(0, this.snapshotIndex + 1)
  }
}

// 撤销
undo(): void {
  if (this.snapshotIndex >= 0) {
    this.snapshotIndex--
    const data = deepCopy(this.snapshotData[this.snapshotIndex]) || []
    // 处理当前选中组件状态
    if (this.curComponent) {
      const needClean = !data.find(component => this.curComponent!.id === component.id)
      if (needClean) {
        this.curComponent = null
        this.curComponentIndex = null
      }
    }
    this.componentData = data
  }
}

// 重做
redo(): void {
  if (this.snapshotIndex < this.snapshotData.length - 1) {
    this.snapshotIndex++
    this.componentData = deepCopy(this.snapshotData[this.snapshotIndex])
  }
}
```

**命令示例**：
- 新增组件：execute → 添加，undo → 删除
- 删除组件：execute → 删除，undo → 恢复
- 修改属性：execute → 更新值，undo → 恢复原值

这就是标准编辑器设计。

### 3.3 状态管理与双向联动

基于 Pinia 实现全局状态管理，属性面板与画布状态实时双向联动：

```
点击画布组件 → 更新 curComponent → 属性面板读取 store → 修改数据 → 自动驱动画布更新
```

**本质**：响应式数据共享。

### 3.4 拖拽系统实现

**核心思想**：把鼠标位移映射成组件坐标变化。

```typescript
// 拖拽核心逻辑
const handleMouseDown = (e: MouseEvent) => {
  const startX = e.clientX
  const startY = e.clientY
  const { left, top } = component.style

  const handleMouseMove = (e: MouseEvent) => {
    const deltaX = e.clientX - startX
    const deltaY = e.clientY - startY

    component.style.left = left + deltaX
    component.style.top = top + deltaY
  }

  document.addEventListener('mousemove', handleMouseMove)
  document.addEventListener('mouseup', () => {
    document.removeEventListener('mousemove', handleMouseMove)
  }, { once: true })
}
```

**需要解决的问题**：
- 画布缩放比例适配
- 元素自动吸附对齐
- 多选拖拽
- 边界限制

### 3.5 图层管理

模拟 CSS 的 z-index 管理，支持：
- 上移一层 / 下移一层
- 置顶 / 置底
- 锁定 / 解锁

**底层本质**：调整数组顺序或修改 z-index 值。

```typescript
// 图层操作实现
upComponent(): void {
  // 上移图层 index，往后移
  if (this.curComponentIndex !== null && this.curComponentIndex < this.componentData.length - 1) {
    swap(this.componentData, this.curComponentIndex, this.curComponentIndex + 1)
    this.curComponentIndex = this.curComponentIndex + 1
  }
}

downComponent(): void {
  // 下移图层 index，往前移
  if (this.curComponentIndex !== null && this.curComponentIndex > 0) {
    swap(this.componentData, this.curComponentIndex, this.curComponentIndex - 1)
    this.curComponentIndex = this.curComponentIndex - 1
  }
}
```

### 3.6 自动对齐

通过几何计算实现：
- 判断组件中心点/边缘差值
- 差值小于阈值（如 5px）时显示辅助线
- 自动吸附对齐

**涉及**：位置信息计算、实时碰撞判断。

### 3.7 版本管理

支持页面版本管理功能：
- **保存版本**：保存当前页面快照，可添加版本名称和描述
- **恢复版本**：一键恢复到历史版本
- **删除版本**：清理不需要的历史版本
- **本地存储**：版本数据持久化到 localStorage

```typescript
// 版本管理实现
interface PageVersion {
  id: string
  name: string              // 版本名称
  description: string       // 版本描述
  snapshot: ComponentData[] // 页面快照
  createdAt: string         // 创建时间
  thumbnail?: string        // 缩略图（base64）
}

saveVersion(name: string, description: string): void {
  const version: PageVersion = {
    id: generateID(),
    name,
    description,
    snapshot: deepCopy(this.componentData),
    createdAt: new Date().toISOString(),
  }
  this.versions.push(version)
  this.saveVersionsToStorage()
}

restoreVersion(versionId: string): void {
  const version = this.versions.find(v => v.id === versionId)
  if (version) {
    this.componentData = deepCopy(version.snapshot)
    this.curComponent = null
    this.curComponentIndex = null
    this.recordSnapshot()
  }
}
```

## 7. 关键业务流程

### 7.1 拖拽添加组件流程

1. 左侧 `ComponentList` 开始拖拽，写入组件索引到 `dataTransfer`。
2. 在 `Home.vue` 的中间画布区域触发 `drop`。
3. 根据索引从 `component-list.ts` 拿组件模板。
4. 计算落点（鼠标位置 - 画布偏移）。
5. 生成唯一 id（使用 nanoid）。
6. 根据当前画布缩放修正尺寸。
7. `store.addComponent` 加入画布。
8. `store.recordSnapshot` 记录历史。

**理解**：拖拽一次 = 模板实例化 + 坐标定位 + 入栈快照。

### 7.2 选中/移动/缩放/旋转流程

由 `Editor/Shape.vue` 负责：

1. 点击组件：设置当前选中组件。
2. 拖动组件：`mousemove` 实时更新 `top/left`。
3. 八点缩放：按拖拽点位计算新宽高和位置。
4. 旋转：按鼠标相对中心点角度变化计算 `rotate`。
5. 鼠标释放后记录快照。
6. 拖动时发 `eventBus.move`，配合标线吸附。

这是低代码编辑器"手感"的核心代码。

### 7.3 框选与组合流程

由 `Editor/index.vue + store.compose` 完成：

1. 在空白画布按下并拖动，生成框选区域。
2. 计算框中命中的组件（排除锁定项）。
3. 多选命中后写入 `areaData`。
4. 点击"组合"时创建 `Group` 组件。
5. 被组合子组件转为 `Group.propValue`。
6. 原组件从画布删除，只保留组合体。

```typescript
// 组合实现
compose(): void {
  if (this.areaData.components.length) {
    const components: ComponentData[] = []
    // 收集所有选中组件
    this.areaData.components.forEach(component => {
      if (component.component !== 'Group') {
        components.push(component)
      } else {
        // 如果要组合的组件中已存在组合，需要先拆分
        const subComponents = component.propValue as ComponentData[]
        components.push(...subComponents)
      }
    })

    // 创建组合组件
    const groupComponent: ComponentData = {
      id: generateID(),
      component: 'Group',
      label: '组合',
      icon: 'qunzu',
      style: { ...this.areaData.style } as ComponentStyle,
      propValue: components,
      animations: [],
      events: {},
      groupStyle: {},
      isLock: false,
      collapseName: 'style',
      linkage: { duration: 0, data: [] },
    }

    createGroupStyle(groupComponent)
    this.addComponent({ component: groupComponent })
    // 删除原组件
    this.areaData.components.forEach(component => {
      this.deleteComponent(this.componentData.findIndex(item => item.id === component.id))
    })
  }
}
```

拆分相反：把 `Group` 子组件恢复到画布，再删除 `Group`。

### 7.4 图层管理流程

`RealTimeComponentList.vue` 按"顶层到低层"显示组件。

支持：

1. 点击图层选中组件
2. 上移/下移
3. 删除

实际操作落在 store 的图层动作并记录快照。

### 7.5 右键菜单流程

`Editor/ContextMenu.vue` 提供：

1. 复制、粘贴、剪切、删除
2. 锁定、解锁
3. 置顶、置底、上移、下移

右键"粘贴"可按鼠标位置粘贴（`paste(true)`），提升编辑效率。

### 7.6 撤销/重做流程（快照机制）

核心思想：每次有效编辑后把 `componentData` 深拷贝压入快照栈。

1. `recordSnapshot`：压栈并截断未来分支
2. `undo`：索引回退，恢复历史状态
3. `redo`：索引前进，恢复后续状态

这是典型编辑器时间线模型。

### 7.7 预览与截图流程

1. 点击预览 -> 弹出 `Preview.vue`。
2. `Preview.vue` 深拷贝当前组件数据，避免污染编辑态。
3. 每个组件通过 `ComponentWrapper` 挂载并执行动画/事件。
4. 若是截图模式，调用 `html-to-image` 生成 PNG 下载。

### 7.8 保存与恢复流程

1. 保存：`Toolbar.save` 把 `componentData` 和 `canvasStyleData` 写到 localStorage。
2. 进入页面时 `Home.restore` 从 localStorage 恢复画布。

这保证了"刷新后设计不丢"。

### 7.9 JSON 导入导出与代码编辑

1. 导出：把 `componentData` 序列化成 JSON 文本并下载。
2. 导入：支持粘贴 JSON 或上传 JSON 文件。
3. Ace 编辑器：可直接编辑当前组件或画布配置 JSON，再提交回 store。

这相当于"可视化模式"和"代码模式"的双通道。

### 7.10 版本管理流程

1. 点击"保存版本" -> 弹出版本信息填写框
2. 填写版本名称和描述 -> 保存到 `versions` 数组
3. 版本数据持久化到 localStorage
4. 点击"恢复版本" -> 从快照恢复页面数据
5. 恢复后自动记录新快照，支持撤销

## 8. 低代码运行态能力（不仅是拖拽）

### 8.1 动画系统

1. 在右侧动画面板给组件添加动画序列。
2. `runAnimation` 按顺序执行动画并支持循环标记。
3. 预览时组件挂载自动执行动画。

动画配置：
```typescript
interface Animation {
  type: string         // 动画类型（如 fadeIn, slideIn 等）
  duration: number     // 持续时间（ms）
  delay: number        // 延迟时间（ms）
  interationNum: number // 执行次数
  infinite: boolean    // 是否无限循环
  applyTo: 'enter' | 'leave'  // 应用时机

```

### 8.2 事件系统

1. `redirect`：跳转 URL
2. `showMessage`：消息提示

在预览态点击组件会读取 `config.events` 并执行对应 handler。

### 8.3 组件联动系统

`common/useOnEvent.ts` 通过事件总线监听：

1. `v-click`

### 8.4 数据请求能力

`utils/request.ts` 支持组件请求：

1. GET/POST/PUT/DELETE
2. 一次请求或轮询请求（series）
3. 参数类型处理
4. 请求次数限制

```typescript
interface RequestConfig {
  method: 'GET' | 'POST' | 'PUT' | 'DELETE'
  data: Record<string, unknown>[]
  url: string
  series: boolean      // 是否定时发送请求
  time: number         // 定时更新时间(ms)
  paramType: '' | 'string' | 'object' | 'array'
  requestCount: number // 请求次数限制，0 为无限
}
```

## 13. 大厂面试常见追问（附高分答法）

### Q1：你这项目最难的是哪块？

答：最难的是编辑器交互一致性。移动、缩放、旋转、框选、组合都在改同一份状态，我通过 store action 统一收敛，并在交互结束时记录快照，保证行为可回退且不互相打架。

### Q2：为什么要做快照而不是只存操作日志？

答：当前项目优先追求实现稳定和复杂度可控，快照方案实现简单、回退可靠。后续若组件量非常大，再考虑"操作日志 + 局部快照"的混合策略优化内存。

### Q3：快照数量有限制吗？

答：有的，限制最多 50 个快照，防止内存溢出。当超过限制时，会移除最早的快照。

### Q4：怎么支持新增组件？

答：按 `component-list.ts` 元数据规范新增组件目录，提供 `Component.vue + Attr.vue`，注册后即可被拖拽、被编辑、被预览。

### Q5：项目离生产化还差什么？

答：
1. 协作能力（多人实时编辑）
2. 云端存储和版本管理
3. 更安全的表达式/脚本执行沙箱
4. 更完善的权限体系和审计日志
5. 更强的测试覆盖和性能监控
6. 组件市场和模板库

### Q6：性能瓶颈会在哪里？

答：主要在大画布高组件量下的重渲染、快照内存、拖拽计算频率。可以通过渲染分层、节流、防抖、虚拟化图层列表、增量快照来优化。

### Q7：这个项目难在哪？（标准回答思路）

答：
1. **页面数据结构设计**：如何用 JSON 描述复杂页面结构，支持嵌套组件、动态属性
2. **操作抽象与历史记录**：命令模式的设计，如何处理连续操作、批量操作
3. **拖拽与坐标系统**：鼠标坐标到画布坐标的转换，缩放比例适配
4. **数据与 UI 实时一致性**：响应式更新、性能优化、避免循环依赖
5. **可扩展组件机制**：组件注册、动态加载、属性配置面板自动生成

### Q8：版本管理是怎么实现的？

答：版本管理本质是快照的持久化存储。保存版本时，把当前 `componentData` 深拷贝存到 `versions` 数组，并持久化到 localStorage。恢复版本时，从快照恢复数据，并自动记录新快照以支持撤销。

### Q9：组合组件是怎么实现的？

答：组合时，把多个组件收集起来，创建一个 Group 组件，把收集的组件存到 `propValue` 字段。同时计算子组件相对于 Group 的相对坐标（`groupStyle`）。拆分时，把子组件恢复绝对坐标，加回画布，删除 Group。

### Q10：TypeScript 给项目带来了什么好处？

答：
1. **类型安全**：编译时发现类型错误，减少运行时 bug
2. **IDE 支持**：完整的智能提示和自动补全
3. **重构能力**：修改类型定义后，所有引用处自动标红
4. **文档化**：类型定义即文档，降低理解成本
5. **可维护性**：大型项目更易于维护和扩展

## 15. 30秒 / 60秒 / 120秒口述模板

### 30秒

我做了一个 Vue3 + TypeScript 低代码可视化编辑器，支持拖拽搭建页面、组件属性编辑、动画事件配置、版本管理、预览和导入导出 JSON。核心是 Pinia 状态驱动和快照撤销重做机制。

### 60秒

这个项目是一个前端低代码编辑器，页面是左中右三栏：左侧组件库和图层，中间画布，右侧属性动画事件。拖拽组件到画布后可以移动、缩放、旋转、组合、锁定。所有状态集中在 Pinia，编辑后进入快照栈支持撤销重做，最多 50 步。工具栏提供预览、截图、保存、版本管理、JSON 导入导出和 Ace 代码编辑。运行态支持动画、点击事件和组件联动，具备可扩展的元数据组件协议。

### 120秒

我把它当一个"可扩展编辑器内核"来做。入口层负责框架和插件注册，Home 层负责编辑器布局编排，Editor 层处理画布级交互，Store 层维护单一状态源。核心难点是多交互统一：拖拽、缩放、旋转、框选组合都要落在同一份组件数据上且可回退，我通过 action 收敛和快照机制解决一致性。组件模型上我定义了统一元数据协议，把样式、动画、事件、联动、请求都抽象成配置，新增组件不需要改编辑器主流程。交付上支持本地保存恢复、版本管理、JSON 导入导出、预览截图和快捷键工作流，后续可以平滑演进到云端协作和生产级低代码平台。
